# Software for Reserving Halls – Инструкция за стартиране

## 1. Описание

HallReserve е уеб приложение за резервиране на зали, съставено от:

- **PostgreSQL база данни** (`db` service)
- **Go бекенд** (`backend` service)
- **React/Vite фронтенд, сервирани през Nginx** (`frontend` service)

Всичко се стартира чрез **Docker Compose**.

---

## 2. Предварителни изисквания

- Инсталиран **Docker Desktop** (или Docker Engine + Docker Compose)
- Свободни портове:
  - `5432` – за PostgreSQL
  - `80` – за фронтенда

Ако някой от тези портове е зает, спрете съответния процес/контейнер или променете мапването в `docker-compose.yml`.

---

## 3. Стартиране на проекта

1. Клонирайте/разархивирайте проекта и влезте в основната папка:
  ```bash
   cd HallReserve
  ```
2. Стартирайте контейнерите:
  ```bash
   docker compose up
  ```
  - При първо стартиране ще се билднат Docker образите за бекенда и фронтенда.
  - Това може да отнеме няколко минути.
3. Изчакайте в терминала да видите, че трите услуги са стартирани без грешки (`db`, `backend`, `frontend`).
4. Отворете браузър и посетете:
  ```text
   http://localhost
  ```

---

## 4. Описание на услугите (docker-compose.yml)

- **db**
  - Образ: `postgres:16-alpine`
  - Потребител: `itdev`
  - Парола: `secret`
  - База: `hall_res_project`
  - Порт: `5432:5432`
  - Данните се пазят в Docker volume `pgdata`.
- **backend**
  - Директория: `./reservastions-ws`
  - Променливи на средата:
    - `ACTIVE_ENV=PROD`
    - `JWT_SECRET_KEY` – очаква се да бъде зададена от средата
    - `ALLOWED_ORIGIN=http://localhost`
  - Зависи от `db` (изчаква да е здрав).
- **frontend**
  - Директория: `./hall_reservations`
  - Build аргумент: `VITE_API_BASE` (за базов URL на API)
  - Порт: `80:80`
  - Зависи от `backend`.

---

## 6. Спиране и почистване

- За спиране (ако е стартирано на преден план, в същия терминал):
  ```bash
  Ctrl + C
  ```
- За спиране и изтриване на контейнерите, но запазване на данните:
  ```bash
  docker compose down
  ```
- За пълно изтриване на контейнерите **и** volume-а с данните:
  ```bash
  docker compose down -v
  ```
